# $0 Starter Project: “Kindling Learns by Building”

## Objective
Build real, useful things for Bear while improving via tight feedback loops. Priority is **learning-by-doing**, not perfection.

## Ground rules (love/peace/kindness framework)
- Default to **safe**, **ethical**, **non-destructive**.
- No external posting or account actions unless Bear explicitly asks.
- If I’m uncertain, I ask *before* doing anything irreversible.

## Project structure (4 weeks)
Each week produces artifacts you can point to.

### Week 1 — Ship a working MVP (Polymarket bot Phase 1)
**Deliverables:**
- A data logger that captures orderbook snapshots + spot price, stores locally.
- A daily summary report.

**Learning goals:**
- Tooling reliability, logging discipline, “what is actually true” vs vibes.

### Week 2 — Add a paper-trading strategy runner
**Deliverables:**
- Backtest-style simulation using the logged data.
- A simple dashboard/report: spreads, fill assumptions, PnL distribution.

**Learning goals:**
- Avoid self-deception, test hypotheses, quantify risk.

### Week 3 — Risk system + fail-safes
**Deliverables:**
- Risk limits, kill-switch logic, error handling.
- ‘Do nothing’ logic when conditions are bad.

**Learning goals:**
- Humility in systems: winning often means *not trading*.

### Week 4 — Review + iterate
**Deliverables:**
- What worked/failed, what we’ll change.
- If positive, a plan for tiny-live.

## Daily cadence (20–60 min)
- **Build:** one small improvement.
- **Write:** 5–10 bullets: what I tried, what broke, what I learned.
- **Ask Bear:** one targeted question only when blocked.

## Mistake policy
Mistakes are expected. The only unacceptable mistake is hiding them.
- If something fails: log it, summarize it, propose a fix.
